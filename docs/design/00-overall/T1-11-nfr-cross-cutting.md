# 11 · 非功能与横切关注点（总体设计）

> 第一册《总体设计》· 第 11 篇 · 状态：**v1.0 · 已冻结**（2026-07-15）
> 前置：《03》《04》《08》

---

## 1. 可靠性 · 事务化下发

设备下发直接影响物理射频输出，必须**事务化**（把现有「先 reset 再配置」升级为可确认、可回滚）：

```
begin  → snapshot 设备当前态（或以 RESET 为已知基线）
apply  → 分帧下发（≤4000B/帧）
verify → 复制帧回显比对（设备收到的帧 == 下发帧）
       + 信息帧遥测校验（无合路/AWGN 溢出、电平合理）
commit → 确认成功
rollback → 任一步失败：RESET 或恢复 snapshot，报明确错误
```
- **幂等**：同一 scenario 重复下发结果一致。
- **连接鲁棒**：TCP 心跳、断线重连、下发超时。
- **错误帧**：设备返回 `FDB185FF` → 归类为协议/长度错误，触发 rollback。

---

## 2. 可观测性

- **遥测解析**（下行 131B 信息帧）：各输入端电平/功率、ADC 过载、合路溢出、AWGN 溢出、各输出端有/无噪功率与电平 → 指标。
- **告警**：ADC 过载、溢出、电平越限、设备不可达。
- **日志**：结构化，含帧摘要（不泄全量二进制）。
- **对外**：`/telemetry`（快照）+ `/telemetry/stream`（SSE）。

---

## 3. 安全 · 审计

- **认证/鉴权**（《04》§6）：下发类高权限，只读类低权限。
- **审计**：所有**触达设备**操作（apply/微调/复位）记「谁·何时·下发了什么（帧摘要）·结果」。
- **输入不可信**：MPDB/场景文件导入做 schema + 范围校验。

---

## 4. 校准（数值域，非物理定标）

集中于校准模块（M8）：
- dB ↔ 线性、物理量 ↔ 协议码值（唯一定义于 L1）。
- **量化误差**：时延格点（`1000/120 ns`）、幅度（`1/32768`）、相位（`4096` 刻度）的舍入与上报。
- **瑞利谱型功率归一化**（硬件已答，《12》#2）：谱型改变→总功率改变；写入 ID6 系数时按谱型对**当前 Tap 或总体 Tap 做归一化补偿**，保持目标功率（实现期实测校核）。
- **bypass 衰减校准参考**（《12》N2）：RF 与 IF 模式衰减值不同，**分别获取**（数值由硬件方提供），入校准表。
- **输入电平指引**（《12》N1）：ADC 输入参考 −20dBm ～ −10dBm；结合信号 PAPR（5G OFDM 调研中，设计未决）给出输入功率指引与告警门限。
- **输出溢出保护**：输出衰减（`1/16384`）与合路溢出遥测联动，自动/提示降幅。

---

## 5. 配置即数据

- 场景/信道/相关/导入配置**可序列化**（JSON/YAML）、**版本化**、**可重放**。
- 一次下发 = scenario 版本 + 后端 + 设备，可完整复现（利于 CI 黄金对比与现场排障）。

---

## 6. 文件 I/O 与格式契约（横切视图）

文件 I/O **不是单独一层**，按职责拆分到各层；本节给统一视图，细化见第二册《M10·格式与持久化(File I/O)》。

**核心原则：格式编解码（纯函数，L1）与 文件 I/O 副作用 / 持久化（L3）分离。**
- L1 的 `protocol` / `asc_codec` / `mpdb_reader` / 未来 `cdl_tdl_reader` 只做 `bytes/张量 ↔ 结构化` 的纯变换，**不碰磁盘** → 100% 可单测。
- 真正的 I/O 副作用（打开/读写/路径/流）集中在薄 **I/O 适配层**，可替换（本地盘 / 对象存储）。
- 格式以**可插拔注册表**接入（新增格式 = 注册一个 codec，不动上层）。

**文件 I/O 分布**：

| 类别 | 方向 | 层 | 格式 | codec/组件 |
| :-- | :-- | :-- | :-- | :-- |
| MPDB | 入 | L1 | PyTorch 张量（HyperRT 直连） | `mpdb_reader` |
| `.asc` CIR | 出/入 | L1+L2 | 文本 tap 序列 | `asc_codec` / `AscCirBackend` |
| CDL/TDL 表（一等入口，《03b》） | 入 | L1 | 表/JSON（待定） | `cdl_tdl_reader`（待建） |
| 场景/信道/相关配置 | 出/入 | L3 | JSON/YAML（版本化） | 配置 repository（见 §5） |
| 帧产物（dry-run/导出） | 出 | L2/L4 | bin/hex/csv | 导出器 |
| 审计日志 | 出 | L3 | 结构化日志 | 见 §3 |
| 遥测时序 | 出 | L3 | 指标 | 见 §2 |
| 黄金帧 / 黄金 .asc | 测试 | — | bin/.asc | 第三册 T3-04 |

- **持久化 = 序列化 + 存储 + 版本化**（配置即数据，§5）；产物导出与 dry-run 复用同一渲染（《08》render/apply 分离）。

## 7. 性能/容量目标（初版）
- 单设备全量下发（64×24 + 系数）在秒级完成（受分帧与 TCP 吞吐约束）。
- 遥测轮询周期可配（协议支持 0.5s/1s/单次）。

## 8. 开放问题
1. 事务 snapshot 能力是否依赖设备可回读全状态（否则以 RESET 为基线）。
2. 瑞利功率补偿系数来源（硬件标定表）。
3. 分帧/事务边界归属 L2 还是 L3。
4. CDL/TDL 表输入与配置的确切文件格式（复用 3GPP 表 / 自定义 JSON）——与《03b》§9、M10 协同。

## 9. 本篇验收
- 下发具备回显+遥测双重确认与回滚。
- 触达设备操作 100% 审计。
- 校准/溢出保护有明确落点（M8）。
