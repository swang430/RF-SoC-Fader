## 1. 概述与定位

自有 GUI 是平台的**主交互前端**，服务信道工程师（主）与 RT/算法研究者（次）。已定决策：

1. **全新设计**——不复用 ChannelEgine 的 `gui.py`（仅集成其引擎微服务，《T1-07》）。
2. **属 L5 客户端层**——只经 **Python SDK / REST** 调用平台，**不直连**引擎或设备。GUI 与第三方走同一 API，能力对等。
3. 本篇只定**顶层概念**（视图结构、设计立场、技术形态）；界面布局、交互细节、组件规格归 **M11-GUI**（T2 功能设计）。

---

## 2. 三条设计立场（顶层原则）

| # | 立场 | 含义 |
| :-- | :-- | :-- |
| S1 | **GUI 是 API 的第一个客户** | 每个界面动作对应一个 REST/SDK 调用；GUI 做不到的第三方也做不到。用 GUI 开发**反向验收 API 完备性**（缺口即 API 缺口）。 |
| S2 | **可视化围绕 canonical model** | PDP/相关热图/栅格图等全部从 schema（《T1-03c》）渲染，与设备无关——dry-run 无设备也能完整预览。R 矩阵在 schema 中物化（`correlation.r*`）正是供此消费。 |
| S3 | **层级入口即导航** | 用户先选「我从哪一层输入」（RT/GCM/CDL/TDL，《T1-03b》§4），界面按层级呈现对应编辑器——把顶层退化链原则直接变成交互结构。 |

## 3. 五个主视图（信息架构）

| 视图 | 承载用例（《T1-02》） | 核心内容 | 主要 API（《T1-04》） |
| :-- | :-- | :-- | :-- |
| **① 场景库** | UC7 | 场景 CRUD、版本历史、标签检索；任一历史版本可重放（配置即数据，《T1-11》§5） | `/scenarios` |
| **② 信道编辑器** | UC1/UC2 | **按层级分入口**（S3）：<br>· RT → MPDB 导入向导（选库→阵列几何/port_map→导入配置→预览）<br>· GCM → 38.901 场景参数面板（经 ChannelEgine 微服务）<br>· CDL/TDL → 表格编辑器（3GPP JSON，《T1-03c》§5.4）| `/imports`、`/scenarios` |
| **③ 可视化台** | UC2/UC3 | PDP 功率时延谱、角度谱、**相关矩阵 R 热图**、8×8 栅格拓扑图（信道对占用/映射）、时变 CIR 播放预览（A 档） | `/scenarios/{id}`（读 model） |
| **④ 下发面板** | UC3/UC4 | 设备连接态、会话创建（选设备+后端 rfsoc/asc）、**dry-run 预览**（帧摘要/.asc）、事务进度（下发→复制帧校验→遥测确认）、失败回滚提示 | `/sessions`、`/devices` |
| **⑤ 遥测仪表盘** | UC5 | 各端口电平/功率实时曲线、ADC 过载/合路/AWGN 溢出**告警**、历史查询 | `/telemetry(/stream)` |

典型主流程（旗舰 UC2）贯穿 ②→③→④→⑤：导入 MPDB → 看 PDP/R 热图确认 → 选后端下发 → 看遥测确认。

## 4. 技术形态（建议 Web，待定档）

**建议：Web 前端**（浏览器访问，由平台 FastAPI 直接服务静态资源）。

| 维度 | Web（建议） | 桌面（PyQt 类） |
| :-- | :-- | :-- |
| 部署 | ✅ 零安装、随平台发布、多人并发访问 | 每机安装、升级分发成本 |
| 与 REST 架构契合 | ✅ 天然（同一 API） | 需内嵌 HTTP 客户端 |
| 可视化生态 | ✅ 丰富（图表/热图/拓扑库） | 可用但生态较窄 |
| 商业形态 | ✅ 现代仪器主流（Web UI 已是趋势） | 传统 |
| 离线单机 | 同机浏览器即可 | ✅ 略强 |

→ 列为本篇唯一待定档项；确认后 M11 按此展开选型（框架/组件库）。

---

## 5. 边界（本篇不做）

- 不定义界面布局/线框/配色/组件规格 → **M11-GUI**。
- 不新增 API——GUI 只消费《T1-04》既有资源；开发中发现的缺口回馈到 API 设计（S1）。
- 不承载设备直连逻辑（一切经 L4）。
