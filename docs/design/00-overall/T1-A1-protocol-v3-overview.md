## 1. 一句话定位

**上位机 ↔ RF-SoC 信道模拟器之间的私有二进制 TLV 帧协议**：上位机用「参数子帧」逐项写设备寄存器级配置，设备以遥测/回显/错误三类帧回应。协议把设备抽象为**一台可编程的 8×8 参数化 TDL 机**——这是理解全协议的心智模型，也是《T1-03b》退化链终点（参数化 TDL 面）的依据。

## 2. 分层结构（三层）

```
┌─ 帧层(Framing) ──────────────────────────────────────────┐
│  header(4B) + length(int16 LE) + payload + tail(AE86)      │
│  5 种帧头按「方向×用途」区分                                │
├─ 子帧层(TLV 参数单元) ───────────────────────────────────┤
│  param_id(1B) + len(1B) + io(1B) + info(1B) + value(NB)    │
├─ 参数语义层 ─────────────────────────────────────────────┤
│  35 个参数 ID，7 个功能族，每参数专属定标（码值↔物理量）   │
└──────────────────────────────────────────────────────────┘
```

### 2.1 帧层：5 种帧头 = 方向 × 用途

| 帧头 | 方向 | 用途 | 长度语义 |
| :-- | :-- | :-- | :-- |
| `FD B1 85 40` | 上→设备 | **控制帧**（N 个参数子帧） | payload ≤ **4000B**，只计子帧总长 |
| `FD B1 85 41` | 设备→上 | **遥测帧**（信息回传） | 固定 131B |
| `FD B1 85 69` | 设备→上 | **复制回显帧**（原控制帧改头回传） | 同原帧 |
| `FD B1 85 FF` | 设备→上 | **错误帧**（长度超限） | 帧头后仅 `01` + 尾 `AE86`（全帧 7B：`FD B1 85 FF 01 AE 86`） |
| `FD B1 85 50` | **设备→上（单向）** | **串口透传帧**（RF 前端上行中继；**上→设备的 RF 透传走控制帧内 ID30/35**，不用此帧头） | 1–255B |

### 2.2 子帧层：统一 4 字节头 + 值

- **io 字节**：高 4bit=输入端(0–7)、低 4bit=输出端(0–7)——寻址 **8×8=64 信道栅格**的机制。
- **info 字节**：通常为多径编号 `0x00–0x17`（对应多径 1–24）。
- **长度特例**：瑞利滤波系数（ID6）长度字段写 0、实跟 1024B——1 字节长度域装不下的唯一逃生门。

## 3. 设备模型（协议隐含的硬件抽象）

```
8 输入端 ──┐                                    ┌── 8 输出端
           │   64 信道（每输入×输出一条）         │   每输出端:
  输入时延  ├──► 每信道: 24 条多径 ──► 合路 ──►──┤   幅度衰减
  (DDR)    │   每径: 延迟/幅度/相位/多普勒/       │   AWGN 注入
           │        瑞利/扫频/分数时延            │   使能
           └── 拓扑可切: 8×8 / 4×8 / 2×8 / 1×8   └── 端口7可作信号源
               对角信道可「路径扩展」至 8×多径
```

## 4. 参数空间：35 个 ID · 7 个功能族

| 功能族 | ID | 要点 |
| :-- | :-- | :-- |
| **全局/控制流** | 1, 13, 22, 14, 15, 33 | 全局使能；参数复位（保留瑞利系数/输出衰减）；硬件复位；遥测节奏（关/0.5s/1s/单次）；回显开关；频偏相位归零（多径同步起相） |
| **每径参数**（核心） | 2, 3, 7, 32, 4, 5+6, 34 | 使能；主时延（0–1050 × **1000/120 ns**）；幅度（**1/32768**）；相位（0–4095 = 2π）；多普勒（±2²⁷ × **1/35.791394133 Hz**）；瑞利使能 + 256 复数频域谱系数（定多普勒谱形）；分数时延（0.25–4.5 ns） |
| **多普勒扫频**（每径） | 23, 17, 18, 19, 20 | 起点/终点/速度/模式（停/折返）/重启——扫频参数**仅重启后生效**（双缓冲语义） |
| **信道/输入级** | 31, 12, 24 | 信道主时延（0–6090 × 50/3 ns）；输入大时延（DDR，最大 6×10⁸ ns 量级，>10 µs 才有效）；支持时延的输入数（默认 4） |
| **输出端** | 10, 11, 8, 9 | 使能；衰减（**1/16384**，不影响 AWGN）；AWGN 使能 + 功率（k=100–65535，单位 1/4096，noise = 1.32×10⁵·k²） |
| **信号源**（端口7） | 25–29 | 使能/频率（1/71.58278827 Hz）/脉冲使能/脉宽/周期（25/3 ns） |
| **RF 前端透传** | 30, 35 | 中继到 J3（收）/J4（发）串口：控频（300 MHz–6 GHz）、三级衰减（0.5 dB 步进）、偏置 |
| **拓扑/扩展** | 21, 16 | 输入拓扑切换（8×8/4×8/2×8/1×8）；对角信道路径扩展（8× 多径，消耗配对信道） |

**单位制特征**：每个参数一套专属定标（时延 1000/120 ns、幅度 1/32768、相位 4096 刻度/圈、多普勒 1/35.79 Hz…），全部小端。这正是平台把「码值换算」集中在 L1（《T1-03》）、schema 只存物理量（《T1-03c》原则 1）的原因。

## 5. 反向通道：遥测帧（131B）

设备按 ID14 设定的节奏主动回传：各输入端**电平×8 + 功率×8 + ADC 过载位图**；各输出端**合路溢出、AWGN 溢出、有/无噪功率×8、电平×8**。这是事务闭环确认（下发后核验无溢出，《T1-11》§1）与 GUI 遥测仪表盘（《T1-14》视图⑤）的数据源。

## 6. 可靠性机制：有什么、缺什么

**协议提供**：
- **复制回显**（ID15）：设备把收到的控制帧原样改头回传 → 上位机比对，确认「设备收到的 = 我发的」；
- **错误帧**：长度超限时回 `FD B1 85 FF 01 AE 86`（7B 全帧）；
- **周期遥测**：间接确认设备运行态。

**协议未提供**（架构事实，平台在上层补偿）：
- **无 CRC/校验和**——帧尾为固定字节，完整性依赖链路层（TCP 的可靠性成为实质依赖，《T1-03》ADR-6）；
- **无序列号/逐帧 ACK**——回显是唯一写确认，无法关联多帧流水（多帧接收语义 = 待硬件确认 Q5）；
- **无参数读回**——不能查询单参数当前值，只有聚合遥测（故事务设计以「RESET 到已知基线」替代「读回快照」，《T1-11》§8-1）；
- **无设备寻址**——一条链路一台设备（多设备由平台 `device_id` 在传输层区分，《T1-10》）；
- **无空间相关/极化字段**——MIMO 相关只能由上层塌缩进 amp/phase（B 档立论依据，《T1-06》；极化传导归约为 XPR，《T1-03c》§5.3）。

## 7. 协议的「性格」与对平台的意义

1. **写单向、寄存器风格**：逐参数写 + 粗粒度回读，无事务概念——平台 L2 的**事务化下发**（reset→写→回显比对→遥测核验→commit/rollback）是对它的补层（《T1-11》§1）。
2. **参数原子、帧内批量**：单帧 payload ≤4000B，64 信道全量配置必须**分帧**——分帧策略与多帧语义是 Q5（《T1-12》）。
3. **TDL 机抽象**：设备可编程面 = delay/amp/phase/doppler + 统计衰落谱形，决定退化链终点（《T1-03b》：一切模型退到 TDL 才能上参数面）。
4. **双缓冲苗头**：扫频「重启后生效」是协议里唯一的影子寄存器语义；未来 CIR 注入（A 档，Q1）若落地，可参考此机制。
5. **实现现状**：现有 `protocol.py` 覆盖 13/35 ID 编码、0/4 种下行帧解码——P0 基座（M1）的补全清单由此而来（《T1-13》）。
