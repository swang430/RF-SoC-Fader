## 1. 概述与定位（双层测试体系）

第二册的每个模块文档都自带「测试设计」节（统一为**类别/内容/判据**三列表）与「本篇验收」节——它们回答「**模块自身对不对**」，**以单元/契约级为主、个别篇含集成级条目**（如 T2-02 §8 环回集成、T2-06 §7 编排契约），已随第二册冻结。

**本册回答的是另一个问题：「功能开发是否处于设计约束之下」。** 测试在本册中不是开发的附属品，而是**独立的监督章程**：

- **向上**，它以三个冻结基线为唯一裁决依据（`design-t1-v1.0` / T1-03c v1.2 / `design-t2-v1.0`）——实现与设计不符时，测试站在设计一边；
- **向下**，它通过追溯矩阵（§4）、黄金基线（§5-A/T3-04）、分层 gate（§7）把每一次代码合并置于监督之下——**没有对应测试义务闭合的功能不算完成**；
- **横向**，它收编第二册测试节未覆盖的跨模块缝隙（§5-E）、系统级行为与硬件在环实测（§6），并为散落各篇的「实测定/验收定」留白建立定标程序（T3-05）。

一句话：**T2 的测试节保证模块正确，T3 保证开发不越出顶层设计。**

## 2. 监督原则（本册宪法，五条）

1. **设计即法**。实现与冻结设计冲突时以设计为准；设计确有错误，先按升版纪律改设计（PR 评审 + 升版，先例 T1-03c v1.1→v1.2），再同 PR 改实现与测试。**禁止改测试迁就实现**——测试代码中任何与冻结文档不符的期望值变更，评审时按设计变更对待。
2. **黄金基线主权**。黄金样本（帧 hex/`.asc`/数值矩阵/OpenAPI）必须有**独立于实现的出处**（协议 docx 示例、ChannelEgine 样例、3GPP 表、解析推导），首录与变更都是设计事件（走 PR、注明出处）——**不得由被测实现生成后自我认证**。
3. **契约测试双向同源**。跨模块契约由双方从同一 fixture 消费（错误表 ↔ SDK 异常映射、OpenAPI ↔ SDK 黄金 diff、NPZ 布局 ↔ 引擎契约、allowed_ops 词表 ↔ GUI 按钮）——单侧改动即 diff 红，防三方漂移（《T2-07》§7、《T2-09》§4/§7 已立先例，本册推广为通则）。
4. **负向路径义务**。设计中每一条「拒绝不猜」「计数上报不静默」「显式报错」条款都必须有对应**负例用例**（能力门拒 A 档/B+、超 24 径显式拒、超范围码值丢弃计数、SchemaMigrationAmbiguous、UncalibratedError 绝不静默按 0……）——T3-02 汇成一张**负向用例矩阵**，正向绿不抵负向缺。
5. **实测台账化**。凡「实测定/实测校核/实测认知」必须挂**《T1-12》编号或 T2 模块开放问题条目**（《T1-12》：Q2/N1/N2/H1–H5；T2 侧：pacing/吞吐 T2-02 §9-3/9-4、遥测标定 T2-08 §8-1），由 HIL（T3-03）兑现；**判定性实测**出 pass/fail 用例，**认知性实测**（如 H5）出记录模板与观察报告——不允许无编号的散落实测承诺。

## 3. 测试金字塔（五层——T3-02 详规）

| 层 | 名称 | 对象与手段 | 归属 |
| :-- | :-- | :-- | :-- |
| L1 | 单元 | 纯函数（编解码/换算/量化合并/校验器），零 I/O | T2 各模块测试节（已冻结）；T3 收编+补缺 |
| L2 | 契约 | 模块间接口：假设备（六剧本+扩展）、假引擎（全端点 stub）、黄金 fixture 双向 | 同上 |
| L3 | 集成 | 进程内多模块编排（三源全桩、状态机矩阵、TCP 环回假设备 render→apply→commit） | T2 个别条目（已冻结：T2-02 §8 环回集成、T2-06 §7 编排契约等）**收编** + 本册详规（T3-02）——收编项在追溯矩阵沿用 T2 出处，不重复建账 |
| L4 | 系统 | 全栈：REST 起、SQLite、假设备/真引擎——UC1–UC8 走通、审计/鉴权/SSE/三前端一致性 | **本册**（T3-02/T3-05） |
| L5 | HIL | 真机 + 射频测量：控制环（HIL-C）与射频测量环（HIL-R），见 §6 | **本册**（T3-03） |

- 《T2-02》§8「环回集成」明示为「T3-03 HIL 的软前身」——L3 的 TCP 环回假设备与 L5 HIL-C 共用剧本集，只换对端。
- 分层不豁免：上层绿不抵下层缺（HIL 通过不豁免契约用例，反之亦然）。

## 4. 追溯矩阵制度（治理核心工具）

**每条设计条款的测试义务可查、每个用例的设计出处可溯。** 矩阵条目：

```
条款（册-篇-节，如 T2-02 §4） → 测试义务（一句话） → 用例 ID → 层（L1–L5） → 状态（未建/在测/绿/豁免）
```

- **用例 ID 命名与存放**归 T3-01；矩阵本体随测试仓库版本化（配置即数据，《T1-11》§5）。
- **变更联动**：冻结文档升版的 PR 必须携带矩阵 diff——受影响用例的增/改/废在同一 PR 内闭合；矩阵中「豁免」状态必须注明批准记录与期限。
- 首版矩阵以 §5 挂账盘点为种子，T3-02/03/04/05 各篇落地时逐条编号。

### 5-B 集成/系统义务（→ T3-02）

假设备六剧本（正常/篡改 1 字节/超时/错误帧/中途断连/遥测溢出位）及其扩展（§5-E）；假引擎全端点 + 熔断/恢复/版本协商；三源编排全桩（调用序①materialize→②reduce→③能力门→④render）；状态机全矩阵（含全部非法迁移与 close 可达性）；双会话设备互斥（非阻塞 DeviceBusy）；重启恢复（ACTIVE→DEVICE_DIRTY、孤儿 op aborted_by_restart）；tweak 三剧本分流；审计完备矩阵（五类判据）；鉴权矩阵（3 scope×全端点）；SSE 断线续传/resync；三前端一致性**两个面**（REST vs SCPI 同 L3 调用序录制断言，T2-07 §7/§10；GUI vs SDK 同语义操作的 HTTP 调用序逐调用等价，T2-11 §6——GUI 不经 SDK、M9 为语义对照物）；隔离性（引擎宕机不影响设备链路）；依赖隔离（主进程无 torch）。

### 5-C HIL 义务（→ T3-03，详见 §6）

P1 真机最小闭环（真实 MPDB→下发→遥测确认无溢出，《T1-13》）；dirty→recover 与 close(disable) 冒烟——T2-06 §10 原文允许**真机或 TCP 环回**二择，本册把真机档收编为 HIL-C 冒烟项（不改 T2-06 判据，环回档仍在 L3 兑现）；遥测 HIL 冒烟（真机物理量合理、过载位人为触发可告警，T2-08 §10）；真引擎冒烟（UMa+CDL-A 端到端，T2-03 §9）；**实测台账全量**——《T1-12》编号项：Q2 E_ref 谱归一定标、N1 输入电平（PAPR）、N2 bypass 衰减标定、H1–H5（B+ 前提与认知项）；T2 开放问题项：pacing/逐帧吞吐（T2-02 §9-3/9-4，其中微帧吞吐并入 H4）、遥测码值→物理量标定对表（T2-08 §8-1）；C1–C4 连续性判据（B+ 预备，《T1-15》§7）。

### 5-D 阈值定标留白（→ T3-05 定标程序）

C2 多普勒谱溅射阈值（场景验收定）；B+ 幅度步长上界（验收定标）；`frobenius_rel_err` 场景验收阈值（量化/截断后不判死）；性能目标（64×24+系数全量下发**秒级**、遥测周期 0.5s/1s/单次可配，《T1-11》§7）。

### 5-E T2 测试节缺口（盘点新发现，→ T3-02 补缺用例）

1. `apply_micro` 微帧通道无专项用例（echo 纪律/失败分流同 apply、asc 后端应拒——T2-06 close(disable)/tweak 的承载面）；
2. 遥测捕获窗竞态（遥测先于请求回显到达）与 cadence-restore echo 异常路径在 T2-02 正文详述但测试表未列剧本；
3. `SerialPassthroughFrame`（0x50 变长帧）解析无用例（T2-01 流式解析表只点名 0x69 畸形前缀）；
4. **T2-02 侧**活性判据（传输层 N 周期无遥测→degraded）与**非事务期**断连重连→dirty 无专项剧本（六剧本仅含事务中断连；M8 侧 device_silent 告警已由 T2-08 §7 覆盖，属另一义务不重复挂账）；
5. 事务失败注入的**阶段×帧序位置二维矩阵**未建：六剧本枚举了失败类型，但 VERIFYING 阶段遥测请求帧 echo 失败（failure=telemetry_req）、遥测等待超时（telemetry_wait）、电平不合理判负（_levels_plausible）、多帧计划「第 k 帧失败」的 FailureInfo 定位、`_rollback` 内 best-effort 恢复周期档失败等**注入位置维**在 T2-02 §8 测试表之外（与六剧本的差集，正文 §4/§5 有行为定义）。

## 6. 硬件在环（HIL）定位（→ T3-03 详规）

### 6.1 两级环

```
HIL-C 控制环（无射频测量）                HIL-R 射频测量环
┌────────────┐   TCP    ┌──────────┐      ┌──────────┐  RF   ┌──────────────┐  RF   ┌──────────────┐
│ CEP 平台    ├─────────►│ RF-SoC   │      │ 信号源    ├──────►│ RF-SoC       ├──────►│ 频谱仪        │
│ (被测系统)  │◄─────────┤ 信道仿真器│      │ (激励)    │       │ 信道仿真器    │       │ (测量)        │
└────────────┘ 回显/遥测 └──────────┘      └────┬─────┘       └──────▲───────┘       └──────┬───────┘
                                               │      SCPI/VISA     │ TCP（CEP 控制）       │
                                               └──────────┤ 测试编排脚本（T3 资产） ├────────┘
```

- **HIL-C**：真机版 TCP 环回——事务/回显/遥测/pacing/吞吐/重连，剧本与 L3 环回集成共用（只换对端）。
- **HIL-R**：**信号源 → 信道仿真器（受 CEP 控制）→ 频谱仪**的物理闭环——验证配置的信道在射频面真实成立。测试编排脚本同时控制 CEP（REST/SDK）与仪器（SCPI/VISA）。注意：此处对仪器的 SCPI 是**测试台作为客户端**，与平台自身的 SCPI 兼容层（M7 对外前端）是两回事。

### 6.2 最小仪器集的可测边界（诚实声明）

以「信号源 + 频谱仪」为基线配置，可直接验证：

| 物理量 | 方法 | 判据来源 |
| :-- | :-- | :-- |
| 幅度/相对功率 | CW 单音 + marker 相对电平（bypass 基准，N2） | 幅度码值定标（1/32768） |
| 多普勒频移 | CW 单音 → 频移读数（**前提：信号源与频谱仪共 10 MHz 参考锁定**——GHz 载频下 0.1 ppm 本振差即数百 Hz，与被测量同量级；频率精度预算归 T3-03） | `Hz = code/35.791394133` |
| 衰落多普勒谱 | CW + 长时间谱密度平均 vs Jakes/平坦理论谱形 | ID6 谱形、C2 |
| AWGN | C/N 谱密度 | AWGN 功率定标（1/4096） |
| **多径时延差** | 双径干涉法：两 tap 配置 → 频域梳状纹波间隔 `Δf = 1/Δτ`（扫频或宽带激励） | 时延定标（1000/120 ns） |
| 幅度瞬态（C3 的 RF 瞬断/包络毛刺） | 频谱仪**零扫宽**（zero-span，幅度-时间）——基线可测 | 《T1-15》C3 幅度分量 |
| 相位跳变/相位瞬态（C1、B+ 相位连续性、C3 相位分量） | **超出频谱仪标量能力**——需矢量信号分析（VSA）/双通道采集，列 T3-03 仪器矩阵开放项 | 《T1-15》C1/C3 |

绝对时延、逐径相位与 MIMO 相关性（R 数值核验，P3 验收）同样需要扩展仪器（VNA/VSA/多通道）——T3-03 建**仪器能力矩阵**：每条 HIL 判据 ↔ 最小仪器需求，基线配置测不了的显式挂账，不降格判据迁就仪器。

### 6.3 安全规程

射频接入前置检查，**两侧都要**：①被测输入侧——电平按 M8 电平指引（ADC −20～−10dBm，N1 PAPR 余量）、bypass 衰减档核对（N2）、遥测过载位监视随测，防 ADC 过载损伤；②仪器侧——频谱仪输入衰减/参考电平按被测输出最大电平（含 AWGN 叠加）预设、核对仪器最大安全输入电平与 DC block 需要性，防仪器前端损伤。规程细则归 T3-03。

## 7. Gate 制度（→ T3-05 详规）

| Gate | 触发 | 内容 | 不过则 |
| :-- | :-- | :-- | :-- |
| PR gate | 每 PR | 改动模块 L1/L2 + 其消费者契约用例 + 追溯矩阵 diff | 不得合并 |
| CI gate | main 每合并 | L1–L3 全量 + 黄金全量 | main 红即修复优先 |
| 发布 gate | 版本发布 | L4 系统 + 黄金 + 性能冒烟（秒级下发） | 不得发布 |
| 验收 gate | 里程碑（P0–P4） | 《T1-13》各阶段验收信号 + HIL 报告（P1 起） | 里程碑不关闭 |

红线：黄金变更必须独立 PR 评审（原则 2）；跳过任何 gate 需书面豁免入追溯矩阵（含期限）。
