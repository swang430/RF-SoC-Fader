## 2. HIL-C 控制环（真机 TCP，无射频测量）

**剧本复用**：L3 环回集成剧本**只换对端**重跑（《T3-00》§3——I-ORCH 段在真机上的重放不另编 ID，矩阵以运行环境区分）；以下为**真机专属** HC 用例：

## 3. HIL-R 射频测量环（信号源 → RF-SoC → 频谱仪）

### 3.1 台架与前置

- 接线：信号源 RF OUT → DUT 输入端口 i；DUT 输出端口 j → 频谱仪 RF IN；**10 MHz 共参考**（信号源 REF OUT → 频谱仪 EXT REF IN，或公共参考分配器）——多普勒/谱形测量的物理前提（《T3-00》§6.2）。
- 编排：测试脚本双客户端——PyVISA 控仪器 + `cep_sdk` 控平台（《T3-00》§6.1；对仪器的 SCPI 是测试台做客户端，与平台 SCPI 兼容层无关）。
- **bypass 基准流程**：每会话开始先走 bypass 档测链路损耗（含线缆），记录为本会话幅度基准（N2 档核对；未标定 → 依赖绝对幅度的用例自动 skip 并留痕，不得按 0 处理——`UncalibratedError` 语义的台架版）。
- 前置检查按 §5 安全规程执行并在报告中打勾留痕。

### 3.2 逐物理量测量方法（HR 用例段）

| ID | 物理量 | 方法（前提/步骤） | 判据面 | 误差预算要点 |
| :-- | :-- | :-- | :-- | :-- |
| HR-CAL-001 | 幅度/相对功率 | CW@fc；bypass 基准 → 设幅度码值序列（大步长点测，如每 6 dB）→ marker delta | 码值定标曲线（1/32768 语义；单步进低于仪器分辨，不测单步） | 仪器幅度不确定度（±0.5 dB 级）；线缆重复接插 |
| HR-CAL-002 | 多普勒频移 | CW；共参考锁定确认 → 设 doppler 码点测集（±100 Hz/±1 kHz/上限档）→ 频移读数（窄 RBW/FFT 模式） | `Hz = code/35.791394133` 点测 | 参考残差（锁定后 mHz 级，可忽略）；RBW 选择 |
| HR-CAL-003 | 衰落多普勒谱 | CW + 瑞利使能（Jakes/平坦）→ 长时间平均功率谱密度 | 谱形对理论（U 形双峰位≈±f_max、平坦带宽）；**C2 溅射阈值的定标数据源**（程序归 T3-05） | 平均次数 vs 谱方差；本振相噪底 |
| HR-CAL-004 | AWGN | CW + AWGN 设定档 → C/N 谱密度测量 | AWGN 功率定标（1/4096 语义） | 噪声带宽修正、检波方式 |
| HR-CAL-005 | 多径时延差 | 双 tap 配置（如 Δτ=10/50/100 bin）；扫频或宽带噪声激励 → 频域梳状纹波间隔 | `Δf = 1/Δτ`（时延定标 1000/120 ns）——大 Δτ 验定标；**1 bin（8.33 ns→纹波周期 120 MHz）受 span/带宽限制**，小 Δτ 边界显式挂账 | 纹波读数取多周期平均；两径幅度不等时纹波深度修正 |
| HR-CAL-006 | 幅度瞬态（C3 幅度分量） | 零扫宽（幅度-时间）+ 触发：分别在 re-apply 与 tweak 时刻观察包络 | re-apply **预期有瞬断**（B 档契约，非缺陷）；tweak 无毛刺 = **H2 判定** | 零扫宽时间分辨/触发抖动 |
| HR-CAL-007 | B+ 幅度步长×溅射扫描（定标专用） | CW + tweak 幅度步长梯度（如 {0.1, 0.2, 0.5, 1, 2, 3} dB）× 每档触发更新 → 谱密度测量（与 HR-CAL-003 **同一谱测量配置**，保证与 C2 阈值可比） | 产出**溅射-步长曲线**（T3-05 §3-2 的定标数据源）；前提=H2 判定通过（HR-CAL-006） | 每档重复取平均；步长档覆盖到明显溅射出现 |

**仪器能力矩阵**（判据 ↔ 最小仪器档；基线测不了的显式挂账、不降格判据——《T3-00》§6.2）：

| 判据 | 基线（信号源+频谱仪） | 扩展档 |
| :-- | :-- | :-- |
| 幅度/多普勒/谱形/AWGN/时延差/幅度瞬态 | ✅ HR-CAL-001…006 | — |
| C1 相位跳变、相位瞬态、B+ 相位连续性 | ❌（标量） | VSA（矢量分析）或双通道采集+下变频 |
| 绝对时延/群时延 | ❌ | VNA |
| MIMO 相关性 R 核验（《T1-13》P3 验收） | ❌ | 双通道相干采集（2×2 需双源双收或开关矩阵，拓扑见 §6-4） |
| H1（改写 ID4 相位积分器行为）**判定** | ❌（基线只能做间接观察：改写 f₁→f₂ 时零扫宽包络无瞬断+谱线连续迁移=倾向性证据，**不构成判定**） | VSA 相位轨迹直接观测 |

### 3.3 《T1-12》台账兑现程序（分型）

- **判定性**（pass/fail 用例 + 定标数据落 `ConfigRepo`，M8/M10 消费）：Q2 E_ref（HR-CAL-003 谱开/关功率差 → 归一化系数实测校核）、N2 bypass（逐档衰减实测入校准表）、H2/H3（HR-CAL-006 tweak/分数时延无毛刺判定 → capabilities 翻转的依据）、H4（HC-M2-010/011 的微帧版——生效延迟/抖动/持续吞吐 → B+ update_rate 上界预算）。
- **认知性**（记录模板，非 pass/fail）：H1（基线只有倾向性证据，判定挂 VSA 扩展档）、H5（HC-M2-020）。模板字段：操作序列（帧级）/仪器设置/观测（trace 引用）/现象描述/结论倾向/对契约的影响评估（预期均为「无」——平台契约已不依赖，《T1-15》Q0）。
- N1（输入电平/PAPR）：属设计输入调研而非台架测量，HIL 仅在 §5 电平预算中消费其当前口径（−20～−10 dBm）。

## 4. HIL 报告模板（P1 验收 gate 交付物，《T3-00》§7/§8）

```
1. 会话头：日期/地点 · 台架配置（仪器型号/固件/共参考锁定确认）· DUT 固件版本 ·
   平台版本（git commit/tag）· 参与人 · 安全前置检查清单（§5，逐项勾选留痕）
2. 用例结果表：HC/HR ID × 结果（pass/fail/skip+原因）× 测量数据引用（trace/截图编号）
3. 定标数据：本会话产出的校准量（bypass 档/E_ref/pacing 建议值/吞吐曲线）
   ——落 ConfigRepo 的引用号（M10），报告只引用不复制
4. 认知观察记录：H1/H5 模板条目（§3.3）
5. 偏差与豁免：与预期不符项、waived 项（批准人/期限，入追溯矩阵）
6. 遗留项：未跑用例与原因、下次会话计划
```

- 归档：报告（markdown）+ 原始测量数据（仪器 trace 导出/截图）入 `docs/hil-reports/<日期>/`，仓库版本化；追溯矩阵状态随报告结果同 PR 联动（`building→green` 或 fail 留案）。

## 5. 安全规程（《T3-00》§6.3 细则）

1. **电平预算表先行**（每会话头核对）：信号源输出上限 → 线损 → DUT 输入落在 −20～−10 dBm（N1 口径，PAPR 余量按当前调研值）；DUT 输出最大电平（含 AWGN 叠加与增益档）→ 频谱仪输入衰减/参考电平预设，校核仪器最大安全输入与 DC block 需要性。
2. **上电顺序**：仪器/负载先在位（DUT 输出端**永不空悬**——接频谱仪或匹配负载）→ DUT 上电 → 平台连接 → 信号源 RF ON 最后。下电逆序。
3. **过载中断规则**：遥测 ADC 过载/输出溢出位触发 → 立即 RF OFF、记录现场、排查后再续（M8 告警在 HIL 会话中为硬中断信号，不是仅告警）。

## 6. 开放问题

1. 扩展档仪器（VSA/VNA/双通道）采购或租用决策——C1 判定、H1 判定与 P3 的 R 核验以此为前提；仪器能力矩阵（§3.2）为采购输入。
2. pacing 扫描粒度与满配剧本的代表性（64 信道全写 vs 分档）——首次 HIL-C 会话按 §2 四档起步，数据回来再细化。
3. 仪器 SCPI 驱动组织：纯 PyVISA 指令串 vs 厂商驱动库——随首次台架搭建定（倾向纯 SCPI 指令表入 `tests/hil/instruments/`，可 fake 化供排练）。
4. 2×2 MIMO 测量拓扑（P3）：双通道相干接收 vs 单通道+开关矩阵分时（牺牲同时性、依赖重放确定性）——随扩展档决策一并定。

## 7. 本篇验收

- HC/HR 用例段 ID 分配完整，逐条挂台账编号或 T2 开放问题号（原则 5 闭环）；
- 六项基线测量方法各有前提/步骤/判据面/误差预算要点，可按 §3.1 直接搭台执行；
- 仪器能力矩阵对「基线测不了」的判据无一降格（C1/H1/R 核验显式挂扩展档）；
- HIL 报告模板六节齐备，P1 验收 gate 可照章执行（《T3-00》§7 时序死结已解）。
