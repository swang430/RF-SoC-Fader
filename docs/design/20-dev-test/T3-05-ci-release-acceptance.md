## 1. 概述与定位

本篇把《T3-00》§7 的四级 gate 落成**可配置的编排**，为 §5-D 的四项阈值留白定**定标程序**（数值由程序产出并走裁决 PR，本篇不拍数值——首版建议值均标注「试运行值」），并把《T1-13》P0–P4 验收信号映射到 gate 项与用例段。

**非职责**：用例内容（→T3-02/T3-03）、HIL 报告模板（→T3-03）、黄金资产（→T3-04）。

## 2. Gate 编排落地

| Gate | 触发 | 编排内容 | 时间预算（试运行值） |
| :-- | :-- | :-- | :-- |
| **PR gate** | 每 PR | ①改动模块的 U/C 段 + 其消费者契约段（矩阵反查改动面）②`check_traceability.py` 四查（《T3-01》§7）③`tools/golden/` 禁 import 静态检查（《T3-04》铁律 1）④矩阵 diff 与代码 diff 对账（人工评审点） | < 5 min（`slow` 标记剪枝） |
| **CI gate**（main 每合并） | merge 后 | L1–L3 全量 + 黄金全量 + coverage 分层报告（M1=100% 强制红线；其余首版目标见 §2.1）+ **治理检查随跑**：`check_traceability.py` 四查（《T3-01》§7 明定进 CI gate——waiver 过期是时间性判定，仅在 PR 时查会让 main 带过期豁免保持绿）+ `tools/golden/` 禁 import 静态检查（《T3-04》铁律 1） | < 20 min |
| **nightly** | 每日 | L4 系统全量（假引擎）+ **真引擎档**（回销《T3-02》§8-1：nightly 跑真引擎、发布/tag 前强制） | 不限 |
| **发布 gate** | 版本发布 | L4 全量 + 黄金全量 + `S-PERF-001` 性能冒烟（阈值 §3-4）+ 发布 checklist（§5） | — |
| **验收 gate** | 里程碑 P0–P4 | §4 映射表逐项 + HIL 报告（P1 起，模板《T3-03》§4）+ 矩阵快照归档 | — |

### 2.1 试运行参数（首版建议值，试运行一个里程碑后走裁决 PR 定版）

- hypothesis CI profile：PR gate `100 examples/用例、deadline 500 ms`；nightly `1000 examples`（回销《T3-01》§9-2 的定数途径）；失败最小化样本归档 `tests/fixtures/shared/hypothesis-regressions/`（转为确定性回归用例）。
- coverage：M1 行覆盖 100%（冻结，T2-01 §8）；L1 纯函数层整体 ≥ 95%；全仓行覆盖 ≥ 85%——低于目标**警告不拦截**（试运行期），定版后转拦截。
- 主干红修复优先：CI gate 红 > 新功能合并（《T3-00》§7「main 红即修复优先」的执行口径——红灯期间仅修复类 PR 可合）。

## 3. 阈值定标程序（《T3-00》§5-D 四项留白）

**定标纪律**：每项定标 = 独立 PR（测量数据引用 + 程序 + 建议值），评审通过后写入 `ConfigRepo`（M10）并回填追溯矩阵；设计文档中「场景验收定/验收定标」字样随定标 PR 回销。

| # | 留白项 | 定标数据源 | 程序 | 试运行值 |
| :-- | :-- | :-- | :-- | :-- |
| 3-1 | C2 多普勒谱重启溅射阈值（《T1-15》§7） | HR-CAL-003（《T3-03》） | 稳态基线谱重复 N 次 → 谱方差分布；更新边界谱与基线差 > 基线方差 × k 判溅射 | k=3、N=20（待台架） |
| 3-2 | B+ 幅度步长上界（《T1-15》§7 机件 4） | **HR-CAL-007**（步长扫描 × 谱溅射测量，《T3-03》§3.2——与 3-1 同一谱测量配置；前提 H2 判定通过） | 溅射-步长曲线 → 取「溅射 < 3-1 阈值」的最大步长 × 0.5 安全系数 | 待 H2 |
| 3-3 | `frobenius_rel_err` 场景验收阈值（《T2-05》§8——量化/截断后不判死、GUI 展示） | **纯软件量化仿真**（无需 HIL）：场景集（UMa/CDL-A/RT 样例）× 24 径截断 + 时延量化 → err 分布 | 按场景取 P95 为「注意线」、P99 为「复核线」（仍不判死，展示口径） | 随 P1 模型集 |
| 3-4 | S-PERF-001 数值（《T1-11》§7「秒级」） | HC-M2-011 吞吐实测曲线（《T3-03》§2） | 全量下发时间基线 → 发布回归线 = 基线 × 1.5；硬上限 = 10 s（「秒级」语义上界） | 硬上限先行，回归线待实测 |

## 4. P0–P4 验收信号映射（《T1-13》→ gate 项）

| 里程碑 | 《T1-13》验收信号 | gate 项 / 用例段 | 签发物 |
| :-- | :-- | :-- | :-- |
| **P0 基座** | 全 35 ID 测试通过；TCP 环回收发一帧并解码回显 | U/C-M1 段全绿 + I-ORCH-001（假设备）绿 + G1/G2 黄金落库 | CI gate 绿 + 矩阵快照 |
| **P1 单信道闭环** | 一条真实 MPDB 链路下发到设备、遥测确认无溢出 | HC-M2-001 + I-ORCH-010 + HC-M1-001（derived 回验） | **首份 HIL 报告**（《T3-03》§4） |
| **P2 引擎+API** | REST 三步 import→scenario→apply 跑通；GUI 可视化 PDP | S-UC1/S-UC2 + C-M11 对应行 + 真引擎冒烟（T2-03 §9） | CI+nightly 绿 + 矩阵快照 |
| **P3 MIMO+高级** | 一条 MIMO 链路复现几何空间相关（对 R 数值核验）；`.asc` 导出被目标设备接收 | 软件面 I-ORCH-010（R 复现 err<1e-10）必须绿；**射频面 R 核验依赖扩展档仪器**（《T3-03》§3.2 矩阵）——仪器未到位时以软件面 + waived（批准/期限）签发；`.asc` 接收为外部记录性验收 | HIL 报告 + waiver 记录 |
| **P4 多设备+SCPI+校准** | 多设备寻址 + SCPI 冒烟；校准/溢出保护验收 | S-UC8 + **SCPI 冒烟路径 = S-UC6-001 的 SCPI 面**（REST vs SCPI 同 L3 调用序一致 + 指令表冒烟，消费 G8 黄金资产——资产本身不是冒烟）+ 定标完成度（N2 bypass/E_ref 入 ConfigRepo、3-1/3-4 定版）+ **溢出保护面**（S-UC5-001 全栈告警链绿 + 真机过载位人为触发告警冒烟《T2-08》§10 + overflow_guard 建议用例 U-M8 段绿——验收信号「校准/溢出保护」两半都有 gate 项） | HIL 报告 + 校准表发布 |

- 每级**准入** = 前级签发完成；每级**签发** = 本级 gate 项全绿（或 waived 留痕）+ 签发物归档。
- 验收 gate 不重复跑测试——它审查的是 gate 记录与矩阵状态（治理审计，非再执行）。

## 5. 发布 checklist（发布 gate 的人工面）

1. 发布物四件对齐：平台包版本 · OpenAPI 快照（G8）· `cep_sdk` 版本（1.x ↔ /v1，T2-09 §6）· 黄金集版本（`fixtures/golden/` 的 tag）；
2. 黄金全量绿 + 追溯矩阵**无过期 waiver**（`check_traceability.py` 第 3 查零红）；
3. S-PERF-001 在 §3-4 线内；
4. CHANGELOG 含设计升版记录（若有）与定标回销记录；
5. 最近一份 HIL 报告的遗留项已评审（不阻塞则留痕）。

## 6. 开放问题

1. coverage 分层阈值定版（§2.1 试运行值跑完 P0–P1 后裁决）。
2. nightly 真引擎档的算力供给（引擎容器 GPU 依赖，随 ChannelEgine 服务层跨仓 PR 一并定，T2-03 §8-4）。
3. 发布节奏：里程碑绑定（P2 首发）vs 时间盒——商用决策，本册只备好 gate。

## 7. 本篇验收

- 四级 gate 各有可执行编排（触发/内容/预算），PR gate 与 CI gate 可在 P0 首日启用；
- 四项阈值留白各有数据源+程序+试运行值，「验收定」不再是无程序的悬账；
- P0–P4 逐级映射到用例段与签发物，验收 gate 是治理审计而非重复执行；
- 发布 checklist 与矩阵/黄金/HIL 报告三方联动闭合。
